# 岗课智评 MVP 产品需求文档

## Problem Statement

职业院校在高水平专业群建设中，需要把课程教学、岗位能力标准、实训任务、评分评价和教学改进连接起来。但现实中，老师设计实训任务、编写评分量规、批改学生提交、分析班级能力短板都很耗时，且评价标准不容易统一。

本项目要做一个面向人工智能应用技术专业群的 AI 实训评分平台，帮助老师用课程资料和岗位标准快速生成分层实训任务与评分量规，让学生完成文本和代码混合提交，并由 AI 给出结构化评分、反馈和能力画像。

## Solution

构建一个名为“岗课智评”的 Next.js 全栈网页 MVP。老师粘贴课程资料和岗位标准后，系统调用 DeepSeek API 生成基础、进阶、挑战三个分层实训任务，并为每个任务生成可编辑评分量规。老师发布任务后，学生端根据历史平均分获得推荐任务难度，提交文本和代码混合内容。系统按评分量规进行 AI 评分，展示总分、维度得分、扣分原因、修改建议和总评语。学生可重新提交并查看前后评分对比。老师端可复核修改评分，并通过班级看板查看成绩列表、能力雷达图、薄弱项分析和 AI 教学建议。

## User Stories

1. 作为老师，我想粘贴课程资料和岗位标准，以便系统理解本次实训的教学目标和岗位能力要求。
2. 作为老师，我想一键生成基础、进阶、挑战三个分层实训任务，以便快速完成差异化实训设计。
3. 作为老师，我想看到每个实训任务的任务目标、任务描述、提交要求和评价重点，以便判断任务是否适合发布。
4. 作为老师，我想让系统为每个任务生成评分量规，以便学生提交后可以按统一标准评分。
5. 作为老师，我想编辑评分量规中的评分点、分值、扣分点和优秀标准，以便保留教师主导权。
6. 作为老师，我想同时发布基础、进阶、挑战三个任务，以便不同能力水平的学生都能获得合适训练。
7. 作为学生，我想选择当前学生身份，以便演示不同学生的历史能力和推荐任务。
8. 作为学生，我想看到系统推荐的任务难度，以便选择适合当前能力水平的实训任务。
9. 作为学生，我想查看基础、进阶、挑战三个任务，以便自由选择要提交的任务。
10. 作为学生，我想在同一个输入框中提交文字说明和代码片段，以便完成 AI 应用技术实训作业。
11. 作为学生，我想看到学术诚信提示，以便明确 AI 辅助学习和个人完成之间的边界。
12. 作为学生，我想提交作业后获得 AI 评分，以便快速知道自己的表现。
13. 作为学生，我想看到总分和五个岗位能力维度得分，以便知道自己的具体短板。
14. 作为学生，我想看到扣分原因和修改建议，以便知道下一次如何改进。
15. 作为学生，我想重新提交修改后的答案，以便形成训练、反馈、改进、再评价的学习闭环。
16. 作为学生，我想看到两次提交之间的总分变化和维度变化，以便明确自己的进步。
17. 作为老师，我想查看学生提交列表和成绩，以便快速掌握班级完成情况。
18. 作为老师，我想查看班级能力雷达图，以便了解班级在岗位能力维度上的整体表现。
19. 作为老师，我想查看薄弱项分析，以便知道下一次课堂要重点讲解哪些内容。
20. 作为老师，我想查看 AI 教学建议，以便调整后续教学和实训安排。
21. 作为老师，我想复核并修改 AI 给出的每个维度分数、扣分原因和总评语，以便保证评价结果可信。
22. 作为演示者，我想应用内置示例数据，以便打开页面即可演示完整流程。
23. 作为演示者，我想在 AI 调用失败时一键载入示例任务和评分量规，以便比赛现场不被网络或 API 波动阻断。
24. 作为演示者，我想页面刷新后数据仍然保留，以便演示过程更稳定。
25. 作为评委，我想看到完整的岗课评闭环，以便判断产品是否真正服务职业教育高水平专业群建设。

## Implementation Decisions

- 产品名为“岗课智评”。
- 专业方向固定为人工智能应用技术专业群。
- MVP 采用比赛展示型工作台，而不是传统多页面教务系统。
- 主界面默认进入 AI 生成实训工作台。
- 页面使用全中文。
- UI 风格采用苹果液态玻璃风：半透明面板、柔和模糊、高光、轻阴影、蓝白银灰科技感。
- 不做真实登录，使用顶部视角切换“老师端 / 学生端”。
- 老师端支持课程资料和岗位标准文本粘贴，不支持 PDF、Word 或图片上传。
- AI 一次生成基础、进阶、挑战三个分层任务。
- 老师可以同时发布三个分层任务。
- 不做题库或历史任务库，只保留当前生成的一组任务。
- 评分量规外层固定为五个岗位能力维度，内层由 AI 根据任务生成具体评分点。
- 五个岗位能力维度和分值为：岗位需求分析 20 分、AI 应用设计 25 分、模型接口调用 25 分、效果评估与调优 20 分、安全合规与文档表达 10 分。
- 学生提交采用文本和代码混合的大输入框。
- 学生端内置班级学生列表，并允许切换当前学生身份。
- 系统根据学生历史平均分推荐任务难度：低于 70 推荐基础，70 到 85 推荐进阶，高于 85 推荐挑战。
- AI 评分结果包含总分、维度得分、扣分原因、修改建议和总评语。
- AI 评分不要求证据引用。
- 学生重新提交后展示完整对比：总分变化、各能力维度变化和改进点变化。
- 老师可以人工复核并修改每个维度分数、扣分原因和总评语。
- 老师端班级看板展示成绩列表、能力雷达图、薄弱项分析和 AI 教学建议。
- 数据使用前端 localStorage 保存。
- 前端内置示例数据，同时允许用户输入覆盖。
- DeepSeek API Key 放在 `.env.local` 中，不暴露给浏览器。
- 后端使用 Next.js API Routes 作为 DeepSeek API 代理。
- AI 失败时展示失败原因，并提供一键载入示例任务和评分量规。
- 不做项目介绍页。
- 不做导出报告。

## Testing Decisions

- 测试以外部行为为主，不测试内部实现细节。
- API 层测试 DeepSeek 代理的结构化输入输出，包括分层任务生成和学生提交评分。
- 业务状态测试覆盖任务生成、评分量规编辑、任务发布、学生提交、重新提交、老师复核和 localStorage 恢复。
- 端到端测试覆盖完整演示路径：老师粘贴资料、生成任务、发布任务、学生提交、AI 评分、重新提交对比、老师查看班级看板。
- 兜底流程测试覆盖 DeepSeek API 失败后的错误提示和示例数据载入。
- 图表和看板测试关注可见行为：成绩列表、雷达图数据、薄弱项分析和教学建议是否随评分数据更新。
- AI 返回内容应通过结构化 JSON schema 或后端解析保护，避免前端因模型输出格式波动崩溃。

## Out of Scope

- 真实账号注册、登录、权限和班级管理。
- PDF、Word、图片等文件解析。
- 真实数据库和服务端持久化。
- 多课程、多班级、多教师协作。
- 题库、历史任务库和任务分类管理。
- 学生文件上传。
- 代码运行、沙箱执行或自动单元测试。
- AI 代写检测、原创性检测和证据引用。
- 报告导出、PDF 导出或打印。
- 项目介绍页、比赛 PPT 或完整申报材料。
- 真实讯飞星火接口适配。

## Further Notes

- MVP 的核心价值不是“自动判卷”本身，而是把岗位标准、课程资料、分层实训、评分量规、能力画像和教学建议连成一个闭环。
- 产品叙事应强调“教师主导、AI 辅助”，因此老师必须能编辑评分量规和复核 AI 评分。
- 现场演示必须保留示例数据兜底，避免 DeepSeek API、网络或 Key 配置问题影响完整演示。
- 后续如要扩展，可加入文件上传、数据库、账号系统、任务库、导出报告和多模型 Provider 配置。
