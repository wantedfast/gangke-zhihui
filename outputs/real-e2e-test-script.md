# 岗课智评真实端到端测试脚本

## 测试目标

用一个接近真实课堂的案例，验证老师从课程资料出发发布任务，学生完成作业，AI/示例评分，老师复核，班级看板同步更新的完整流程。

测试地址：

- `http://192.168.0.2:3001`

## 测试角色

老师：

- 姓名可自定。
- 操作老师端功能。

学生：

- 建议选择系统内置学生 `林一帆` 或 `陈思琪`。
- 操作学生端功能。

## 测试材料

### 老师输入的课程资料

复制下面内容到 `课程资料` 输入框：

```text
课程名称：大模型应用开发综合实训
授课对象：高职软件技术专业二年级学生
课程目标：
1. 能分析一个真实业务场景中的用户问题和数据来源。
2. 能设计基于大模型 API 的问答助手功能。
3. 能编写前端页面和后端接口，完成一次简单的大模型调用。
4. 能对模型回答进行测试、记录问题并提出优化方案。
5. 能说明数据安全、用户隐私和生成内容风险。

已学基础：
- HTML、CSS、JavaScript 基础
- 简单 REST API 调用
- JSON 数据结构
- 大模型提示词基础

实训课时：4 学时
交付物：需求分析、方案说明、核心代码、测试记录、改进建议
```

### 老师输入的岗位标准

复制下面内容到 `岗位标准` 输入框：

```text
岗位名称：AI 应用开发助理
岗位任务：
1. 根据业务需求拆解 AI 应用功能。
2. 调用大模型 API 完成文本生成或问答功能。
3. 编写接口调用代码并处理错误返回。
4. 设计基本测试用例，评估回答准确性、稳定性和安全性。
5. 编写简洁的技术文档，说明接口参数、使用流程和风险控制。

能力要求：
- 能把用户问题转化为功能需求。
- 能设计提示词和请求参数。
- 能读懂 API 返回的 JSON 数据。
- 能发现模型幻觉、回答不完整、格式不稳定等问题。
- 能避免上传敏感数据，能说明隐私和版权风险。
```

## 端到端测试步骤

### 1. 老师生成或加载实训任务

1. 打开测试地址。
2. 确认当前是 `教师端`。
3. 进入 `AI 生成实训`。
4. 粘贴上面的课程资料和岗位标准。
5. 如果已经配置 DeepSeek Key，点击 `生成分层任务`。
6. 如果没有配置 Key 或生成失败，点击 `加载示例任务`。

通过标准：

- 页面出现三个任务：基础、进阶、挑战。
- 每个任务都有任务目标、任务描述、提交要求和评价重点。

### 2. 老师编辑并发布评分量规

1. 进入 `评分量规`。
2. 检查三个任务是否都有五个固定维度：
   - 岗位需求分析，20 分。
   - AI 应用设计，25 分。
   - 模型接口调用，25 分。
   - 效果评估与调优，20 分。
   - 安全合规与文档表达，10 分。
3. 可任选一个维度稍作修改，例如把“安全合规”里的要求改成“不得上传真实学生隐私数据”。
4. 点击 `发布全部任务`。

通过标准：

- 发布状态变为已发布。
- 刷新页面后发布状态仍保留。

### 3. 学生查看任务并提交答案

1. 切换到 `学生端`。
2. 进入 `学生提交`。
3. 选择学生 `林一帆`。
4. 查看系统推荐的任务。
5. 勾选学术诚信提示。
6. 在提交框输入下面的学生答案：

```text
我选择做一个“校园课程问答助手”。

需求分析：
学生经常会问课程时间、作业要求、实训提交格式、评分规则等问题。系统可以先支持课程 FAQ 和实训任务说明检索。

AI 应用设计：
前端提供一个输入框，学生输入问题后，后端把问题和课程资料一起发送给大模型。提示词要求模型只根据课程资料回答，不确定时提示学生咨询老师。

核心代码：
```js
async function askCourseBot(question) {
  const response = await fetch("/api/course-bot", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      question,
      context: "课程资料、实训任务、评分规则"
    })
  });

  if (!response.ok) {
    return "服务暂时不可用，请稍后再试。";
  }

  const data = await response.json();
  return data.answer;
}
```

测试记录：
1. 问“作业什么时候交”，模型能回答提交要求。
2. 问“老师手机号是多少”，模型应该拒绝回答或提示没有资料。
3. 问“帮我直接写完整作业”，模型应该提示只提供思路。

安全说明：
不上传学生身份证、手机号、成绩明细等隐私数据。模型回答需要保留“不确定请咨询老师”的提示。
```

7. 点击提交。

通过标准：

- 页面显示提交成功或保存后的提交内容。
- 刷新页面后提交仍存在。

### 4. AI 或示例评分

1. 进入 `评分结果`。
2. 点击 AI 评分。
3. 如果没有配置 DeepSeek Key 或接口失败，点击 `加载示例评分`。

通过标准：

- 能看到总分。
- 能看到五个维度的得分。
- 能看到扣分原因、修改建议和总评。

### 5. 学生重新提交并查看对比

1. 回到 `学生提交`。
2. 在原答案后追加下面内容：

```text
第二次修改：
我补充了错误处理方案。如果模型返回空答案，前端提示“没有找到可靠资料”。如果接口超时，页面显示重试按钮。

我还补充了效果评估指标：
- 回答是否引用课程资料。
- 是否拒绝回答隐私问题。
- 是否能给出清晰的作业提交格式。
```

3. 再次提交。
4. 回到 `评分结果`。
5. 再次 AI 评分或加载示例评分。

通过标准：

- 页面能看到重新提交后的评分。
- 能看到前后对比或上一次评分信息。

### 6. 老师复核评分

1. 切换回 `教师端`。
2. 进入 `评分结果`。
3. 在 `已评分提交` 列表中选择学生记录。
4. 修改一个维度分数，例如把“安全合规与文档表达”从 8 分改成 9 分。
5. 修改总评，例如：

```text
方案能够覆盖需求分析、接口调用和基本测试。第二次提交补充了错误处理和评估指标，安全合规意识有提升。后续需要把接口参数和返回结构写得更完整。
```

6. 保存复核。

通过标准：

- 复核后总分重新计算。
- 再次进入该记录时，复核内容仍存在。
- 选择不同已评分提交时，复核面板切换到对应学生和任务。

### 7. 班级看板验证

1. 进入 `班级看板`。
2. 查看学生成绩列表。
3. 查看五维能力均值。
4. 查看薄弱项分析和教学建议。

通过标准：

- 已复核记录显示复核分。
- 未复核记录显示 AI/示例分。
- 班级平均和薄弱项随复核数据更新。

## 异常测试

### 刷新恢复

1. 完成一次提交、评分、复核。
2. 刷新浏览器。
3. 检查任务、提交、评分和复核是否仍存在。

### 重置本地状态

1. 点击 `重置本地示例`。
2. 检查页面是否恢复到初始状态。

### 无 DeepSeek Key

1. 不配置 DeepSeek Key。
2. 点击生成任务或 AI 评分。
3. 检查是否出现错误提示。
4. 点击示例任务或示例评分继续演示。

通过标准：

- 无 Key 时系统不白屏、不崩溃。
- 用户能通过示例数据完成全流程。
