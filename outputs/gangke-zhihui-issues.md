# 岗课智评 MVP 开发任务拆分

## 1. 搭建 Next.js 工作台骨架

**Type:** AFK  
**Blocked by:** None  
**User stories covered:** 22, 24, 25

## What to build

搭建可运行的 Next.js 全栈应用骨架，提供“岗课智评”主工作台、老师/学生视角切换、左侧导航、苹果液态玻璃风基础视觉、内置示例数据和 localStorage 持久化基础。

## Acceptance criteria

- [ ] 应用可通过一个本地开发命令启动。
- [ ] 首屏默认进入 AI 生成实训工作台。
- [ ] 顶部可切换老师端和学生端。
- [ ] 左侧导航包含核心模块：AI 生成实训、评分量规、学生提交、评分结果、班级看板。
- [ ] 页面整体为全中文苹果液态玻璃风。
- [ ] 内置示例数据可在首次打开时加载。
- [ ] 刷新页面后主要状态可从 localStorage 恢复。

## 2. 实现 AI 生成分层实训任务闭环

**Type:** AFK  
**Blocked by:** 1  
**User stories covered:** 1, 2, 3, 4, 6, 23

## What to build

老师粘贴课程资料和岗位标准后，可调用后端代理请求 DeepSeek 生成基础、进阶、挑战三个分层实训任务和对应评分量规。AI 失败时显示原因，并可一键载入示例任务和评分量规。

## Acceptance criteria

- [ ] 老师端提供课程资料和岗位标准文本输入区。
- [ ] 点击生成后请求后端 API，而不是在浏览器直接暴露 API Key。
- [ ] 成功后展示基础、进阶、挑战三个任务卡片。
- [ ] 每个任务包含任务目标、任务描述、提交要求和评价重点。
- [ ] 每个任务同时生成五个岗位能力维度的评分量规。
- [ ] 失败时展示错误原因。
- [ ] 失败状态下可一键载入示例任务和评分量规。
- [ ] 生成结果保存到 localStorage。

## 3. 实现评分量规编辑与发布

**Type:** AFK  
**Blocked by:** 2  
**User stories covered:** 4, 5, 6, 21

## What to build

老师可以编辑每个任务的评分量规，并同时发布基础、进阶、挑战三个任务。评分量规固定围绕五个岗位能力维度，总分 100。

## Acceptance criteria

- [ ] 每个任务都展示评分量规编辑区域。
- [ ] 五个维度固定为：岗位需求分析、AI 应用设计、模型接口调用、效果评估与调优、安全合规与文档表达。
- [ ] 分值固定为 20、25、25、20、10。
- [ ] 老师可以编辑每个维度的评分点、优秀标准、合格标准和扣分点。
- [ ] 老师可以发布全部三个任务。
- [ ] 发布状态保存到 localStorage。

## 4. 实现学生身份、任务推荐与提交

**Type:** AFK  
**Blocked by:** 3  
**User stories covered:** 7, 8, 9, 10, 11

## What to build

学生端内置班级学生列表，允许切换当前学生。系统根据学生历史平均分推荐基础、进阶或挑战任务。学生可以选择任务并提交文本和代码混合内容。

## Acceptance criteria

- [ ] 学生端可选择当前学生。
- [ ] 每个学生有历史平均分和基础能力画像示例数据。
- [ ] 低于 70 分推荐基础任务，70 到 85 分推荐进阶任务，高于 85 分推荐挑战任务。
- [ ] 学生可以查看三个已发布任务。
- [ ] 推荐任务有明显但不夸张的视觉标识。
- [ ] 学生提交区支持文本和代码混合输入。
- [ ] 学生提交页展示学术诚信提示。
- [ ] 提交内容保存到 localStorage。

## 5. 实现 AI 评分与学生反馈

**Type:** AFK  
**Blocked by:** 4  
**User stories covered:** 12, 13, 14, 15, 16, 23

## What to build

学生提交后，系统通过后端代理调用 DeepSeek，根据任务和评分量规输出结构化评分。学生可查看总分、维度得分、扣分原因、修改建议和总评语，并可重新提交查看前后对比。

## Acceptance criteria

- [ ] 学生提交后可触发 AI 评分。
- [ ] 评分 API 输入包含任务、评分量规、学生身份和学生提交内容。
- [ ] 评分结果包含总分、五个维度得分、扣分原因、修改建议和总评语。
- [ ] AI 返回格式异常时前端不崩溃，并给出可理解提示。
- [ ] 学生可以重新提交。
- [ ] 重新提交后展示总分变化、维度得分变化和改进点变化。
- [ ] 评分记录保存到 localStorage。
- [ ] AI 失败时可使用示例评分结果兜底。

## 6. 实现老师复核评分

**Type:** AFK  
**Blocked by:** 5  
**User stories covered:** 17, 21

## What to build

老师端可以查看学生提交和 AI 评分结果，并复核修改每个维度分数、扣分原因和总评语。修改后的结果成为看板统计依据。

## Acceptance criteria

- [ ] 老师端展示学生提交列表、提交状态和当前分数。
- [ ] 老师可以查看单个学生的评分详情。
- [ ] 老师可以修改每个维度得分。
- [ ] 老师可以修改扣分原因和总评语。
- [ ] 修改后总分重新计算或保持一致性提示。
- [ ] 复核结果保存到 localStorage。

## 7. 实现班级能力看板

**Type:** AFK  
**Blocked by:** 6  
**User stories covered:** 17, 18, 19, 20, 25

## What to build

老师端基于学生评分记录展示班级成绩列表、能力雷达图、薄弱项分析和 AI 教学建议，让评委看到“岗课评”闭环。

## Acceptance criteria

- [ ] 看板展示班级学生成绩列表和任务完成状态。
- [ ] 看板按五个岗位能力维度计算班级平均分。
- [ ] 看板展示能力雷达图或等价的能力可视化。
- [ ] 系统根据最低维度生成薄弱项分析。
- [ ] 系统生成简短 AI 教学建议。
- [ ] 老师复核评分后，看板数据同步更新。

## 8. 完成 DeepSeek 配置、兜底与验证

**Type:** AFK  
**Blocked by:** 2, 5, 7  
**User stories covered:** 22, 23, 24, 25

## What to build

补齐 `.env.example`、DeepSeek Provider 配置说明、结构化 JSON 保护、兜底示例路径和基本验证脚本，确保 MVP 可稳定演示。

## Acceptance criteria

- [ ] 项目包含 `.env.example`，说明 DeepSeek API Key、Base URL 和模型名。
- [ ] 未配置 API Key 时，页面给出清晰提示。
- [ ] 生成任务和评分两个 API 都有错误处理。
- [ ] 示例数据路径可完整跑通老师端和学生端演示。
- [ ] localStorage 清空后仍可重新载入示例数据。
- [ ] 提供最少的 lint 或构建验证命令，并通过验证。
- [ ] 最终 README 或页面提示说明本地启动方式。
